**Overview**: Configuration root defining target hosts and global workstation variables.

## STRUCTURE
- `hosts.yml`: Main inventory file defining host groups and connection
  parameters.
- `remote_hosts.yml`: Secondary inventory for alternative remote targets.
- `group_vars/all.yml`: Canonical configuration root containing all `vps_`
  prefixed variables.
- `deploy_full.yml`: Playbook-style inventory definition for full stack
  deployment.

## CONVENTIONS
- **Variable Prefix**: All global workstation variables MUST use the `vps_`
  prefix.
- **YAML Standard**: Use 2-space indentation and multi-line map syntax for
  readability.
- **Ordering**: Keep variables in `all.yml` logically grouped by functional
  category (System, Security, Desktop, etc.).
- **Hostname Compliance**: `vps_hostname` must be RFC 1123 compliant.
- **Security Level**: Control hardening intensity via `vps_security_level`
  (standard|hardened|paranoid).

## ANTI-PATTERNS
- **Plaintext Secrets**: NEVER store passwords, keys, or hashes in plaintext
  within this directory.
- **Environment Specifics**: Avoid hardcoding IP addresses in `hosts.yml`;
  prefer using `setup.sh` overrides.
- **Duplicate Definitions**: Variables defined in `group_vars/all.yml` should
  not be repeated in role defaults.
- **Manual Edits**: Do not modify inventory files during a running deployment
  to avoid race conditions.
